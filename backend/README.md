# iKlippa Backend & ML Engine — Technical Documentation

Welcome to the backend architecture of iKlippa! This repository contains the Go API Gateway and the Python Machine Learning Engine. 

This README explains the purpose of every backend file and provides the exact API endpoints the Frontend needs to interact with.

---

## 🏗 Architecture Overview

iKlippa uses a 3-tier backend microservice architecture:
1. **Go API Gateway (`:8080`)**: The main entry point. Handles frontend traffic and orchestrates the other services.
2. **IBM Granite / Ollama (`:11434`)**: Runs locally. Takes prompts and generates video scripts.
3. **Python ML Engine (`:8000`)**: Runs NLP sentiment analysis, searches stock media APIs, and predicts virality.

---

## 📂 File Explanations

### `backend/` (Go Gateway)
*   **`main.go`**: The core Gin web server. Exposes the main `POST /api/director/generate` route. It takes the prompt, asks the local LLM to write the script, forwards that script to the Python ML engine, and finally combines everything into one payload.
*   **`services/watsonx.go`**: An HTTP client that wraps the local Ollama API. It sends the user's prompt to the `granite-code:3b` model and extracts the generated text.

### `ml/` (Python ML Engine)
*   **`app.py`**: A FastAPI server that exposes the `POST /analyze` endpoint. It wraps all the ML scripts into a single REST API so the Go gateway can easily communicate with it.
*   **`train.py`**: The XGBoost training script. Connects to `data/` to train the virality prediction model.
*   **`model.json` & `feature_schema.json`**: The exported weights and input contract for the trained XGBoost model.
*   **`scripts/collect_yt_data.py`**: Fetches raw training data (views, tags, duration) via the YouTube Data API v3.
*   **`scripts/parse_script.py`**: The NLP engine. Uses `spaCy` to extract noun chunk keywords, and `VADER` to analyze the sentiment/mood of the generated script.
*   **`scripts/stock_search.py`**: Hits the Pexels API to find 5 royalty-free HD video clips based on the NLP keywords.
*   **`scripts/music_search.py`**: Hits the Jamendo API to find royalty-free background music based on the NLP mood label.

## 🔌 API Endpoints (For Frontend Integration)

### 1. The Director Endpoint (Main)
This is the **only** endpoint the React frontend needs to call to get a complete video package.

**Request:**
*   **URL:** `POST http://localhost:8080/api/director/generate`
*   **Content-Type:** `application/json`
*   **Body:**
```json
{
  "prompt": "Write a 15-second cinematic documentary script about exploring a terrifying abandoned hospital in the woods at night."
}
```

**Response (200 OK):**
*   Returns a massive combined JSON payload.
*   `script`: The raw string of text generated by IBM Granite. Use this for subtitles/text rendering.
*   `ml_data`: The metadata extracted by Python, including pre-fetched royalty-free media links ready for the WebCodecs pipeline.
```json
{
  "script": "(The camera pans over a dark, desolate forest...)",
  "ml_data": {
    "keywords": ["desolate forest", "abandoned hospital"],
    "mood": {
      "compound": -0.9704,
      "label": "suspense"
    },
    "pacing": {
      "avg_words_per_sentence": 15.8,
      "label": "slow"
    },
    "stock_videos": [
      {
        "id": 4323944,
        "duration": 6,
        "link": "https://videos.pexels.com/video-files/4323944/...mp4"
      }
    ],
    "background_music": [
      {
        "id": "12345",
        "name": "Dark Cinematic Ambience",
        "duration": 120,
        "audio": "https://prod-1.storage.jamendo.com/?trackid=12345&format=mp32"
      }
    ]
  }
}
```

### 2. The Direct ML Endpoint (Optional Bypass)
If you already have a script and just want the NLP metadata + stock media *without* asking the AI to write a new one, you can bypass Go and hit the Python server directly.

**Request:**
*   **URL:** `POST http://localhost:8000/analyze`
*   **Content-Type:** `application/json`
*   **Body:**
```json
{
  "script_text": "Your custom script text here..."
}
```

**Response (200 OK):**
*   Returns exactly the `ml_data` object shown above.
